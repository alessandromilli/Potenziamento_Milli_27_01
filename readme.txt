Sei uno sviluppatore dell'azienda ospedaliera ASL 27 .

In una cartella del server deve esserci un file CSV strutturato con i seguenti 2 campi:

Username;

password.

Nella medesima cartella sono memorizzati i referti dei pazienti sotto forma di file txt. Il nome del file txt è strutturato nel seguente modo: aaaa-mm-cccccccc.txt dove:

• aaaa è un numero di 4 cifre che identifica l'anno del referto;

• mm è un numero di 2 cifre che identifica il mese del referto;

• cccccccc è un numero di 8 cifre che identifica il codice del paziente (ultime 8 cifre della tessera

sanitaria).



Occorre realizzare un'applicazione C/S che consenta all'utente, previa autenticazione con username e password attraverso il file CSV che contiene le credenziali, di ottenere la lista dei referti di un certo anno di pubblicazione con un dato codice del paziente (l'anno di riferimento e il codice del paziente sono scelti/inseriti dall'utente); la lista dei referti può essere ottenuta utilizzando uno qualsiasi dei controlli che ritieni opportuno.

Se l'autenticazione dell'utente non va a buon fine, la ricerca non potrà essere effettuata.



È possibile creare l'applicazione secondo la propria ipotesi/idea.



È obbligatorio consegnare come risposta un file allegato zip che contiene:

1. tutti i file contenenti il codice che è stato prodotto

2. un file di testo che contiene la DOCUMENTAZIONE del lavoro svolto



Documentazione: la documentazione consente di valutare la "proprietà" del codice consegnato inteso come la competenza di saper scegliere consapevolmente del codice già scritto e adattarlo per una nuova situazione oppure la scrittura consapevole di codice ex novo.
